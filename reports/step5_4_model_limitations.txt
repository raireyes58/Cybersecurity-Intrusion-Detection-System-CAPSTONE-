MODEL LIMITATIONS ANALYSIS
============================================================

1. CLASS IMBALANCE
   Original dataset class ratio (Step 2): 1.24:1
   Mitigation applied: SMOTE oversampling (Step 4.2) on training data only.

   RISK: SMOTE generates synthetic minority-class examples by interpolating
   between real neighbours — it does not add genuinely new information about
   rare attack patterns, only more balanced exposure to the patterns already
   present. If a genuinely novel attack type is under-represented in the
   original data, SMOTE cannot manufacture examples of it.

   The test-set F1_macro (0.7778) reflects performance on the TRUE,
   still-imbalanced distribution — this is the only honest measure of
   real-world reliability; training-set metrics on SMOTE-balanced data
   would overstate performance.

2. DATA LEAKAGE RISK
   Deliberate leakage-prevention steps taken in this notebook:
   - Identifier columns (e.g. session_id) dropped in Step 3.1, before any
     encoding — prevents the model from memorising row identity.
   - RobustScaler (Step 4.1) fit on X_train ONLY, then applied to X_test —
     fitting on combined data would leak test-set statistics into training.
   - SMOTE (Step 4.2) applied to X_train ONLY — applying it to test data
     would inflate metrics with synthetic, non-real evaluation examples.

   REMAINING RISK: this dataset's session-level features (e.g. session_duration,
   failed_logins) are aggregated per-session. If any sessions in the raw data
   originate from the same underlying user/device across the train/test split,
   subtle user-level leakage could still exist that a random row-level split
   would not catch — this was not testable with the fields available.

3. OVERFITTING RISK
   Train-test F1_macro gap        : +0.0154
   Cross-validation F1_macro (4.10): 0.7931 ± 0.0071

   ✅ The train-test gap is within a reasonable range, suggesting the model
   is not substantially overfit to the training data.

   ✅ Low cross-validation variance supports that the single test-set result
   is representative and not a fold-dependent artifact.

SUMMARY
   This model's reported 0.7778 F1_macro should be read as a realistic,
   but not infallible, estimate — bounded by known imbalance-correction
   trade-offs, mitigated-but-not-eliminated leakage risk, and quantified
   overfitting/stability checks above.
